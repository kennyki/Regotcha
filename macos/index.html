# Beware of file name case sensitivity

If you're using the common Mac OS Extended (Journaled) file system, keep in mind that it's case insensitive but case preserving.

For example, when importing a file manually (with a tool like Grunt) a typo mistake will work locally but fail in UNIX environment - which is common for deployment setup.

// "Croppie" should be "croppie"
"<%= settings.dir.node_modules %>/Croppie/croppie.js"
